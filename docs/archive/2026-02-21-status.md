## ❌ What's Missing

### Critical Gaps

1. **Grouped aggregation only supports `sum`** — `min`, `max`, `mean`, `count` silently produce wrong results when used with `groupby()`
2. **Filter doesn't remove rows** — multiplying by a boolean mask zeroes values but keeps row count, corrupting `mean` and any downstream logic
3. **No `.count()` expression** — needed for group-by
4. **String/date columns silently dropped** — group-by key columns (strings) never appear in output
5. **No GPU device path** — everything hardcoded to `DeviceRef.CPU()`

### Missing Plan Nodes

- `Sort` (order by)
- `Join` (inner, left, right, outer)
- `Limit` (top N)
- `Distinct` (unique rows)

### Missing Expression Operations

- `.count()` — row count / non-null count
- `.std()`, `.var()` — standard deviation, variance
- `&` (and), `|` (or), `~` (not) — boolean combinators
- `.is_null()`, `.fill_null()` — null handling
- `.cast()` — type casting
- String ops (`.str.contains()`, `.str.len()`, etc.)
- Date ops (`.dt.year()`, `.dt.month()`, etc.)

### Missing Mojo Kernels

- `group_min`, `group_max`, `group_mean`, `group_count` — needed for grouped aggregation
- Generic scatter/gather kernels for true filtering
- Sort kernel
- Join kernel (hash join or sort-merge)
